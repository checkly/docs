The `checkly deploy` command deploys all your checks and associated resources like alert channels to your Checkly account. This command synchronizes your local monitoring-as-code configuration with your Checkly account.

<Accordion title="Prerequisites">
Before using `checkly deploy`, ensure you have:

- A Checkly CLI project initialized with `npx checkly init`
- At least one check or resource defined in your project
- Valid Checkly account authentication (run `npx checkly login` if needed)
- A `checkly.config.ts` or `checkly.config.js` configuration file

For additional setup information, see [CLI overview](/cli/overview).
</Accordion>

## Basic Usage

The basic command deploys all resources to your Checkly account, synchronizing your local monitoring-as-code configuration.

```bash Terminal
npx checkly deploy
```

## Extended Usage

```bash Terminal
npx checkly deploy [options]
```

## Unique Options

<ResponseField name="--preview, -p" type="boolean">
Preview the differences between your local configuration and your Checkly account without making any changes.

### Usage
```bash Terminal
npx checkly deploy --preview
npx checkly deploy -p
```

### Examples
```bash Terminal
# Preview all changes
npx checkly deploy --preview

# Preview with specific config file
npx checkly deploy --preview --config ./checkly.staging.config.ts

# Preview with verbose output
npx checkly deploy -p --verbose
```

Shows you exactly what will be added, modified, or removed during deployment.
</ResponseField>

<ResponseField name="--output, -o" type="boolean">
Show applied differences after deploying, providing a summary of what was changed.

### Usage
```bash Terminal
npx checkly deploy --output
npx checkly deploy -o
```

### Examples
```bash Terminal
# Deploy and show changes
npx checkly deploy --output

# Deploy with force and show output
npx checkly deploy --force --output

# Combine with other options
npx checkly deploy -o --verbose
```
</ResponseField>

<ResponseField name="--debug-bundle" type="boolean">
Generate a JSON file containing the data sent to Checkly servers when deploying. Useful for debugging configuration issues.

### Usage
```bash Terminal
npx checkly deploy --debug-bundle
```

### Examples
```bash Terminal
# Generate debug bundle
npx checkly deploy --debug-bundle

# Generate bundle without actually deploying
npx checkly deploy --preview --debug-bundle

# Debug with verbose output
npx checkly deploy --debug-bundle --verbose
```

Creates a `checkly-bundle-debug.json` file in your project directory.
</ResponseField>

<ResponseField name="--no-schedule-on-deploy" type="boolean">
Prevent checks from running automatically when they are deployed.

### Usage
```bash Terminal
npx checkly deploy --no-schedule-on-deploy
```

### Examples
```bash Terminal
# Deploy without scheduling
npx checkly deploy --no-schedule-on-deploy

# Deploy to staging without scheduling
npx checkly deploy --config ./checkly.staging.config.ts --no-schedule-on-deploy

# Force deploy without scheduling
npx checkly deploy --force --no-schedule-on-deploy
```

Useful when you want to deploy changes but delay execution until later.
</ResponseField>

## Common Options

<ResponseField name="--config, -c" type="string">
Specify a particular configuration file to use instead of the default `checkly.config.ts` or `checkly.config.js`.

### Usage
```bash Terminal
npx checkly deploy --config <file-path>
npx checkly deploy -c <file-path>
```

### Examples

<Tabs>
<Tab title="Different Environments">

```bash Terminal
# Deploy to staging
npx checkly deploy --config ./checkly.staging.config.ts

# Deploy to production
npx checkly deploy --config ./checkly.production.config.ts
```

</Tab>
<Tab title="Custom Config Location">

```bash Terminal
# Use config from different directory
npx checkly deploy --config ./configs/checkly.custom.js

# Relative path config
npx checkly deploy -c ../shared-configs/monitoring.config.ts
```

</Tab>
</Tabs>

**Default**: Looks for `checkly.config.ts` or `checkly.config.js` in the current directory.
</ResponseField>

<ResponseField name="--force, -f" type="boolean">
Skip confirmation dialogs and proceed with the operation automatically. Especially useful in CI/CD environments.

### Usage
```bash Terminal
npx checkly deploy --force
npx checkly deploy -f
```

### Examples

<Tabs>
<Tab title="CI/CD Deployment">

```bash Terminal
# Automated CI deployment
npx checkly deploy --force

# Output
Deploying without confirmation...
✓ 12 checks deployed
✓ 3 alert channels updated
```

</Tab>
<Tab title="With Other Options">

```bash Terminal
# Force deploy with preview and output
npx checkly deploy --force --preview --output

# Force deploy to staging
npx checkly deploy -f --config ./checkly.staging.config.ts
```

</Tab>
</Tabs>

**Warning**: Use with caution as this bypasses safety prompts.
</ResponseField>

<ResponseField name="--env, -e" type="string">
Pass environment variables to the deployment process. Variables passed this way will override any existing variables stored in your Checkly account.

### Usage
```bash Terminal
npx checkly deploy --env <KEY=value>
npx checkly deploy -e <KEY=value>
```

### Examples

<Tabs>
<Tab title="Single Variable">

```bash Terminal
# Deploy with environment URL
npx checkly deploy --env ENVIRONMENT_URL="https://staging.acme.com"

# Output
Deploying with environment variables...
✓ ENVIRONMENT_URL overridden for this deployment
```

</Tab>
<Tab title="Multiple Variables">

```bash Terminal
# Deploy with multiple environment variables
npx checkly deploy --env API_KEY="staging123" --env BASE_URL="https://api.staging.com"

# Combined with config file
npx checkly deploy -e VERSION="v2.1.0" --config ./checkly.production.config.ts
```

</Tab>
</Tabs>

**Usage**: You can pass multiple environment variables by using the flag multiple times.
</ResponseField>

<ResponseField name="--verbose, -v" type="boolean">
Show detailed logs and output for all operations, including full deployment information.

### Usage
```bash Terminal
npx checkly deploy --verbose
npx checkly deploy -v
```

### Examples

<Tabs>
<Tab title="Standard Deployment">

```bash Terminal
# Standard deploy
npx checkly deploy

# Output
✓ 12 checks deployed
✓ 3 alert channels updated
```

</Tab>
<Tab title="Verbose Deployment">

```bash Terminal

```

</Tab>
</Tabs>

**Use cases**: Debugging deployments, detailed CI logs, troubleshooting configuration issues.
</ResponseField>

## Git Integration

When you deploy a project, you can attach Git-specific information so changes to any resources are displayed in the Checkly web UI with the correct commit, branch, and author information.

The following Git variables are automatically detected from your local or CI environment:

- `CHECKLY_GIT_REPO_URL` - Repository URL
- `CHECKLY_GIT_BRANCH` - Current branch
- `CHECKLY_GIT_COMMIT_SHA` - Commit hash
- `CHECKLY_GIT_COMMIT_MESSAGE` - Commit message
- `CHECKLY_GIT_COMMIT_AUTHOR_NAME` - Author name
- `CHECKLY_GIT_COMMIT_AUTHOR_EMAIL` - Author email

You can also set these manually as environment variables if automatic detection doesn't work in your environment.
