The `checkly trigger` command executes checks that are already deployed in your Checkly account. Unlike the `test` command, `trigger` works with any checks in your account regardless of how they were created - via CLI constructs, the web UI, or Terraform.

<Accordion title="Prerequisites">
Before using `checkly trigger`, ensure you have:

- Valid Checkly account authentication (run `npx checkly login` if needed)
- At least one check deployed in your Checkly account
- Appropriate permissions to run checks in your account

For additional setup information, see [CLI overview](/cli/overview).
</Accordion>

## Usage

The basic command triggers all checks in your account that are already deployed.

```bash Terminal
npx checkly trigger [options]
```

## Unique Options

<ResponseField name="--test-session-name" type="string">
Provide a custom name for recorded test sessions when using `--record`.

### Usage
```bash Terminal
npx checkly trigger --record --test-session-name "Custom session name"
```

### Examples
```bash Terminal
# Record with custom session name
npx checkly trigger --record --test-session-name "Pre-deployment validation"

# Record with release information
npx checkly trigger --record --test-session-name "Release v2.1.0 validation"

# Combine with tags and location
npx checkly trigger --tags production --record --test-session-name "Prod health check" --location eu-west-1
```

If not specified, a default name with timestamp and Git information will be used.
</ResponseField>

## Common Options

<ResponseField name="--record" type="boolean">
Record test results in Checkly as a test session with full logging, videos, and traces.

### Usage
```bash Terminal
npx checkly trigger --record
```

### Examples

<Tabs>
<Tab title="Basic Recording">

```bash Terminal
npx checkly trigger --record

# Output

✓ 15 checks triggered from account
```

</Tab>
<Tab title="With Session Name">

```bash Terminal
# Record with custom name
npx checkly trigger --record --test-session-name "Production health check"

# Output
Test session recorded: Production health check
✓ 15 checks triggered from account
```

</Tab>
</Tabs>

**Benefits**: Persistent results, detailed debugging information, team collaboration.
</ResponseField>

<ResponseField name="--tags, -t" type="string">
Filter checks using comma-separated tags. Only checks with all specified tags will be triggered.

### Usage
```bash Terminal
npx checkly trigger --tags <tag1,tag2>
npx checkly trigger -t <tag1,tag2>
```

### Examples

<Tabs>
<Tab title="Production Checks">

```bash Terminal
# Trigger production checks only
npx checkly trigger --tags production

# Output
Triggering 8 checks tagged with: production
✓ Homepage production check
✓ API production health
```

</Tab>
<Tab title="Critical API Checks">

```bash Terminal
# Trigger checks with both critical AND api tags
npx checkly trigger --tags critical,api

# Output
Triggering 3 checks tagged with: critical,api
✓ Payment API check
✓ User authentication API
```

</Tab>
</Tabs>

**Multiple filters**: Use multiple `--tags` flags for OR logic.
</ResponseField>

<ResponseField name="--location, -l" type="string" default="us-east-1">
Run checks against a specified location instead of the default `us-east-1`.

### Usage
```bash Terminal
npx checkly trigger --location <location>
npx checkly trigger -l <location>
```

### Examples

<Tabs>
<Tab title="EU Location">

```bash Terminal
# Trigger from EU West region
npx checkly trigger --location eu-west-1

# Output
Triggering checks from location: eu-west-1
✓ Latency: 45ms (EU optimized)
```

</Tab>
<Tab title="Multiple Regions">

```bash Terminal
# Test different regions in sequence
npx checkly trigger --location us-west-1 --record --test-session-name "US West validation"
npx checkly trigger --location ap-southeast-2 --record --test-session-name "APAC validation"
```

</Tab>
</Tabs>

**Available locations**: `us-east-1`, `us-west-1`, `eu-west-1`, `eu-central-1`, `ap-northeast-1`, `ap-southeast-1`, `ap-southeast-2`, and more.
</ResponseField>

<ResponseField name="--reporter, -r" type="string">
Specify one or more output formats for test results. Useful for different environments and integrations.

### Usage
```bash Terminal
npx checkly trigger --reporter <format>
npx checkly trigger -r <format>
```

### Examples

<Tabs>
<Tab title="CI Integration">

```bash Terminal
# CI-friendly output
npx checkly trigger --reporter ci

# Output
[PASS] Homepage check (850ms)
[FAIL] Payment API (timeout after 30s)
[PASS] User dashboard (1.2s)
```

</Tab>
<Tab title="JSON for Automation">

```bash Terminal
# JSON output for parsing
npx checkly trigger --reporter json --tags production

# Output
{"total":12,"passed":11,"failed":1,"results":[{"name":"Homepage","status":"passed"}]}
```

</Tab>
</Tabs>

**Available reporters**: `list` (default), `dot`, `ci`, `github`, `json`
</ResponseField>

## Key Differences from `checkly test`

| Feature | `checkly test` | `checkly trigger` |
|---------|---------------|------------------|
| **Source** | Local project files | Deployed checks in account |
| **File patterns** | Supports file matching | Not applicable |
| **Check creation** | Can create checks from code | Uses existing checks only |
| **Snapshots** | Can update snapshots | Cannot update snapshots |
| **Dependencies** | Requires local project | Works independently |

## Use Cases

### Pre-deployment Validation
Run your deployed checks before promoting code to production:

```bash Terminal
npx checkly trigger --tags staging --record --test-session-name "Pre-prod validation"
```

### Scheduled Testing
Integrate with CI/CD for scheduled check execution:

```bash Terminal
npx checkly trigger --tags nightly --reporter ci --timeout 1800
```

### Environment-specific Testing
Test deployed checks against different environments:

```bash Terminal
npx checkly trigger --env ENVIRONMENT_URL="https://preview-branch-123.vercel.app"
```
