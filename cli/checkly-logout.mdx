The `checkly logout` command signs out of your current Checkly account and removes stored authentication tokens from your local machine.

<Accordion title="Prerequisites">
Before using `checkly logout`, ensure you have:

- Currently authenticated with Checkly CLI (check with `npx checkly whoami`)
- Access to the terminal/command line

No internet connection is required as logout works locally.
</Accordion>

## Usage

The basic command logs out of your current Checkly account and removes stored authentication tokens.

```bash Terminal
npx checkly logout
```

## What Happens During Logout

### Session Invalidation
The logout process will:
1. **Invalidate your session** on Checkly servers
2. **Remove local tokens** from your machine
3. **Require re-authentication** for future CLI usage

### Token Removal
The command removes authentication tokens stored locally in:

- **macOS**: `~/Library/Application Support/checkly-cli/`
- **Linux**: `~/.config/checkly-cli/`
- **Windows**: `%APPDATA%\checkly-cli\`

## After Logout

Once logged out, you'll need to run [`checkly login`](/cli/checkly-login) before using other CLI commands that require authentication.

Commands that require authentication include:
- `checkly deploy`
- `checkly test --record`
- `checkly trigger`
- `checkly env`
- `checkly import`

## Use Cases

### Security Best Practice
Log out when using shared or public computers:

```bash Terminal
npx checkly logout
```

### Account Switching
Log out before switching to a different account:

```bash Terminal
npx checkly logout
npx checkly login
```

### CI/CD Cleanup
Include logout in CI/CD scripts for security:

```bash Terminal
# Deploy checks
npx checkly deploy

# Clean up authentication
npx checkly logout
```
