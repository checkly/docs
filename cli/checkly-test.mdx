The `checkly test` command executes all checks in your project on the Checkly cloud infrastructure. This command provides a dry-run capability for testing your monitoring setup before deployment, with support for different environments and configurations.

<Accordion title="Prerequisites">
Before using `checkly test`, ensure you have:

- A Checkly CLI project initialized with `npx checkly init`
- At least one check file in your project directory
- Valid Checkly account authentication (run `npx checkly login` if needed)
- A `checkly.config.ts` or `checkly.config.js` configuration file

For additional setup information, see [CLI overview](/cli/overview).
</Accordion>

## Usage

The basic command runs all checks in your project as a test run without deploying them to your account as monitors.


```bash Terminal
npx checkly test [file-patterns] [options]
```



<Tabs>
<Tab title="Unique Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `--list` | Display all checks without executing them | boolean | No |
| `--grep, -g` | Run checks matching a regular expression pattern | string | No |
| `--update-snapshots, -u` | Update golden images and reference snapshots | boolean | No |
| `file-patterns` | Specify file patterns to run specific subsets | string[] | No |

</Tab>
<Tab title="Common Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `--record` | Record test session | boolean | No |
| `--reporter, -r` | Specify output format | string | No |
| `--tags, -t` | Filter by tags | string | No |
| `--verbose, -v` | Show detailed output | boolean | No |

</Tab>

| Option | Description | Type | Required |
|--------|-------------|------|----------|

| `--env, -e` | Pass environment variables | string | No |
| `--env-file` | Load variables from file | string | No |

## Unique Options

<ResponseField name="--list" type="boolean">
Display all checks in your project without executing them. Useful for verifying which checks would be included in a test run.

### Usage
```bash Terminal
npx checkly test --list
```

### Examples
```bash Terminal
# List all checks
npx checkly test --list

# List checks with file patterns
npx checkly test api --list

# List checks with tags
npx checkly test --tags production --list
```
</ResponseField>

<ResponseField name="--grep, -g" type="string">
Run only checks where the check name matches a JavaScript regular expression pattern.

### Usage
```bash Terminal
npx checkly test --grep <pattern>
npx checkly test -g <pattern>
```

### Examples
```bash Terminal
# Run checks matching a specific pattern
npx checkly test --grep "api.*test"

# Run checks with multiple patterns (OR logic)
npx checkly test -g "homepage|checkout"

# Combine with other options
npx checkly test --grep "api" --tags production
```
</ResponseField>

<ResponseField name="--update-snapshots, -u" type="boolean">
Update the golden images and reference snapshots for visual comparison and snapshot tests.

<Warning>Only use `--update-snapshots` when you've verified that visual changes are intentional.</Warning>

### Usage
```bash Terminal
npx checkly test --update-snapshots
npx checkly test -u
```

### Examples
```bash Terminal
# Update all snapshots
npx checkly test --update-snapshots

# Update snapshots for specific checks
npx checkly test api --update-snapshots

# Update snapshots with verbose output
npx checkly test -u --verbose
```
</ResponseField>

<ResponseField name="file-patterns" type="string[]">
Specify file patterns as arguments to run specific subsets of checks. Supports multiple patterns with OR logic.

### Usage
```bash Terminal
npx checkly test <file-pattern> [<file-pattern>...]
```

### Examples
```bash Terminal
# Run specific file
npx checkly test home.spec.js

# Run files containing 'api'
npx checkly test api

# Run files containing 'product' OR 'checkout'
npx checkly test product checkout

# Combine with options
npx checkly test api --record --location eu-west-1
```
</ResponseField>

## Common Options

<ResponseField name="--record" type="boolean">
Record test results in Checkly as a test session with full logging, videos, and traces.

### Usage
```bash Terminal
npx checkly test --record
```

### Examples

<Tabs>
<Tab title="Basic Recording">

```bash Terminal
# Record basic test session
npx checkly test --record

# Output
Test session recorded: https://app.checklyhq.com/test-sessions/abc123
✓ All checks passed
```

</Tab>
<Tab title="With Custom Name">

```bash Terminal
# Record with custom session name
npx checkly test --record --test-session-name "Pre-deployment validation"

# Output
Test session recorded: Pre-deployment validation
✓ All checks passed
```

</Tab>
</Tabs>

**Benefits**: Persistent results, detailed debugging information, team collaboration.
</ResponseField>

<ResponseField name="--reporter, -r" type="string">
Specify one or more output formats for test results. You can combine multiple reporters.

### Usage
```bash Terminal
npx checkly test --reporter <format>
npx checkly test -r <format>
```

### Examples

<Tabs>
<Tab title="JSON Reporter">

```bash Terminal
# JSON output for CI/CD
npx checkly test --reporter json

# Output
{"results":[{"name":"Homepage check","status":"passed","duration":1200}]}
```

</Tab>
<Tab title="CI Reporter">

```bash Terminal
# CI-friendly format
npx checkly test --reporter ci

# Output
[PASS] Homepage check (1.2s)
[PASS] API health check (0.8s)
```

</Tab>
</Tabs>

**Available reporters**: `list` (default), `dot`, `ci`, `github`, `json`
</ResponseField>

<ResponseField name="--tags, -t" type="string">
Filter checks using comma-separated tags. Checks must contain all specified tags to be selected.

### Usage
```bash Terminal
npx checkly test --tags <tag1,tag2>
npx checkly test -t <tag1,tag2>
```

### Examples

<Tabs>
<Tab title="Single Tag Filter">

```bash Terminal
# Run production checks only
npx checkly test --tags production

# Output
Running 8 checks tagged with: production
✓ Homepage check
✓ API health check
```

</Tab>
<Tab title="Multiple Tags">

```bash Terminal
# Run checks with both production AND api tags
npx checkly test --tags production,api

# Output
Running 3 checks tagged with: production,api
✓ API health check
✓ User API check
```

</Tab>
</Tabs>

**Multiple filters**: Use multiple `--tags` flags for OR logic.
</ResponseField>

<ResponseField name="--verbose, -v" type="boolean">
Show detailed logs and output for all operations, including full check execution logs.

### Usage
```bash Terminal
npx checkly test --verbose
npx checkly test -v
```

### Examples

<Tabs>
<Tab title="Standard Output">

```bash Terminal
# Standard test run
npx checkly test

# Output
✓ Homepage check (1.2s)
✓ API health check (0.8s)
```

</Tab>
<Tab title="Verbose Output">

```bash Terminal
# Verbose test run
npx checkly test --verbose

# Output
✓ Homepage check (1.2s)
  - Browser: chromium
  - Location: us-east-1
  - Screenshots: 3 captured
✓ API health check (0.8s)
  - Response time: 245ms
  - Status code: 200
```

</Tab>
</Tabs>

**Use cases**: Debugging, detailed CI logs, troubleshooting check failures.
</ResponseField>
