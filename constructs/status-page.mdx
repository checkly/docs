Use StatusPage to create public status pages that show the uptime of your services through organized cards. Status pages help communicate service availability to your users and stakeholders.

<Info>
Learn more about status pages in the [Status Pages documentation](/docs/communicate/status-pages/).
</Info>

## `StatusPage` Structure

<CodeGroup>

```ts Basic Example
import { StatusPage, StatusPageService } from 'checkly/constructs'

const apiService = new StatusPageService('api-service', {
  name: 'API Service'
})

new StatusPage('company-status', {
  name: 'Company Status',
  url: 'company-status',
  cards: [
    {
      name: 'Core Services',
      services: [apiService]
    }
  ]
})
```

```ts Advanced Example
import { StatusPage, StatusPageService } from 'checkly/constructs'

const apiService = new StatusPageService('api-endpoints', {
  name: 'API Endpoints'
})

const webService = new StatusPageService('web-app', {
  name: 'Web Application'
})

const databaseService = new StatusPageService('database', {
  name: 'Database'
})

new StatusPage('acme-status', {
  name: 'A.C.M.E Status',
  url: 'acme-status',
  customDomain: 'status.acme.com',
  logo: 'https://acme.com/logo.png',
  redirectTo: 'https://acme.com',
  favicon: 'https://acme.com/favicon.ico',
  defaultTheme: 'DARK',
  cards: [
    {
      name: 'Core Infrastructure',
      services: [apiService, databaseService]
    },
    {
      name: 'User-Facing Services',
      services: [webService]
    }
  ]
})
```

</CodeGroup>

## Options

<Tabs>
<Tab title="Unique Options">

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `name` | `string` | ✅ | - | Name of the status page |
| `url` | `string` | ❌ | - | Subdomain under "checkly-status-page.com" (unique across all accounts) |
| `customDomain` | `string` | ❌ | - | Custom domain (e.g., "status.example.com") |
| `logo` | `string` | ❌ | - | URL to logo image for the header |
| `redirectTo` | `string` | ❌ | - | URL to redirect when logo is clicked |
| `favicon` | `string` | ❌ | - | URL to favicon image |
| `defaultTheme` | `string` | ❌ | `'LIGHT'` | Theme: `'DARK'` \| `'LIGHT'` \| `'AUTO'` |
| `cards` | `array` | ✅ | - | Array of card objects containing services |

</Tab>
</Tabs>

## Unique Options

<ResponseField name="name" type="string" required>
Name of the status page displayed in the header and browser title.

### Usage
```ts
new StatusPage('company-status', {
  name: 'Company Status Page'
})
```

### Examples

<CodeGroup>

```ts Company Branding
const companyStatus = new StatusPage('acme-status', {
  name: 'ACME Service Status' // Company brand visible
})
```

```ts Service-Specific
const apiStatus = new StatusPage('api-status', {
  name: 'API Platform Status' // Platform-specific status page
})
```

```ts Regional Status
const regionStatus = new StatusPage('us-east-status', {
  name: 'US East Region Status' // Regional service status
})
```

</CodeGroup>

**Use cases**: Brand visibility, service identification, regional status pages, internal team pages.
</ResponseField>

<ResponseField name="url" type="string">
Subdomain under "checkly-status-page.com" that must be unique across all Checkly accounts.

### Usage
```ts
new StatusPage('company-status', {
  name: 'Company Status',
  url: 'company-status' // Creates company-status.checkly-status-page.com
})
```

### Examples

<CodeGroup>

```ts Company Subdomain
const companyPage = new StatusPage('acme-status', {
  name: 'ACME Status',
  url: 'acme-corp-status' // acme-corp-status.checkly-status-page.com
})
```

```ts Service-Specific URL
const apiPage = new StatusPage('api-platform-status', {
  name: 'API Platform Status',
  url: 'api-platform' // api-platform.checkly-status-page.com
})
```

```ts Team-Specific URL
const teamPage = new StatusPage('frontend-team-status', {
  name: 'Frontend Team Status',
  url: 'frontend-team-internal' // frontend-team-internal.checkly-status-page.com
})
```

</CodeGroup>

**Use cases**: Public status pages, team-specific status pages, service-specific monitoring, temporary status pages.
</ResponseField>

<ResponseField name="cards" type="array" required>
Array of card objects that organize services into logical groups on the status page.

### Usage
```ts
new StatusPage('company-status', {
  name: 'Company Status',
  cards: [
    {
      name: 'Core Services',
      services: [apiService, webService]
    }
  ]
})
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | `string` | ✅ | Display name for the card |
| `services` | `StatusPageService[]` | ✅ | Array of services to display on this card |

### Examples

## Custom Domains

To use a custom domain for your status page:

<CodeGroup>

```bash DNS Configuration
# Add a CNAME record pointing to Checkly
status.example.com CNAME checkly-status-page.com
```

```ts SSL Certificate
// After configuring DNS, verify your domain through the Checkly UI to enable SSL certificates.
// This process ensures secure HTTPS access to your status page.
```

```ts Verification
new StatusPage('custom-domain-status', {
  name: 'Custom Domain Status',
  customDomain: 'status.example.com', // Must be verified
  cards: [
    {
      name: 'Services',
      services: [service]
    }
  ]
})
```

</CodeGroup>


<Warning>
**Domain Verification**: Custom domains must be verified through the Checkly UI before they can be used. DNS changes may take time to propagate.
</Warning>

<Info>
**Service Reuse**: A single `StatusPageService` can be used across multiple status pages and cards, making it easy to maintain consistent service definitions.
</Info>

<Info>
**Logo Requirements**: Use publicly accessible URLs for logos and favicons. Recommended formats are PNG, SVG, or JPG.
</Info>
