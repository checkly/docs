The `checkly env` command manages global environment variables in your Checkly account. Create, update, list, export, and remove variables that are available across all checks in your account.

<Accordion title="Prerequisites">
Before using `checkly env`, ensure you have:

- Valid Checkly account authentication (run `npx checkly login` if needed)
- Appropriate permissions to manage environment variables in your account
- Editor or admin role for creating/updating variables

For additional setup information, see [CLI overview](/cli/overview).
</Accordion>

## Usage

The basic command structure uses subcommands to manage environment variables.

```bash Terminal
npx checkly env <subcommand> [arguments] [options]
```

## Subcommands

| Subcommand | Description |
|------------|-------------|
| `pull` | Export variables to a local file |
| `ls` | List all global variables |
| `add` | Create a new variable |
| `update` | Update an existing variable |
| `rm` | Remove a variable |

## `pull`

Export global variables from your Checkly account to a local file.

### Usage
```bash Terminal
npx checkly env pull [filename]
```

### Options

<Tabs>
<Tab title="Unique Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `filename` | Target file path | string | No |
| `--force, -f` | Overwrite existing file without confirmation | boolean | No |

</Tab>
<Tab title="All Global Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `--config, -c` | Specify a configuration file | string | No |
| `--verbose, -v` | Show detailed output | boolean | No |

For detailed information about global options, see [global options](/cli/global-options).

</Tab>
</Tabs>

### Examples
```bash Terminal
# Pull to default .env file
npx checkly env pull

# Pull to specific file
npx checkly env pull .env.production

# Force overwrite existing file
npx checkly env pull .env.local --force
```

## `add`

Create a new global environment variable.

### Usage
```bash Terminal
npx checkly env add <key> <value> [options]
```

### Options

<Tabs>
<Tab title="Unique Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `key` | Variable name | string | Yes |
| `value` | Variable value | string | Yes |
| `--locked, -l` | Make variable private to read-only users | boolean | No |
| `--secret, -s` | Store as encrypted secret | boolean | No |

</Tab>
<Tab title="All Global Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `--config, -c` | Specify a configuration file | string | No |
| `--verbose, -v` | Show detailed output | boolean | No |

For detailed information about global options, see [global options](/cli/global-options).

</Tab>
</Tabs>

### Examples
```bash Terminal
# Add regular variable
npx checkly env add ENVIRONMENT "production"

# Add locked variable
npx checkly env add INTERNAL_API_URL "https://internal.api.com" --locked

# Add secret variable
npx checkly env add API_SECRET "super-secret-key" --secret

# Add both locked and secret
npx checkly env add DATABASE_URL "postgres://prod-db:5432/app" --locked --secret
```

## `update`

Update an existing global environment variable.

### Usage
```bash Terminal
npx checkly env update <key> <value> [options]
```

### Options

<Tabs>
<Tab title="Unique Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `key` | Variable name to update | string | Yes |
| `value` | New variable value | string | Yes |
| `--locked, -l` | Make variable locked/private | boolean | No |
| `--secret, -s` | Convert to secret (cannot revert) | boolean | No |

</Tab>
<Tab title="All Global Options">

| Option | Description | Type | Required |
|--------|-------------|------|----------|
| `--config, -c` | Specify a configuration file | string | No |
| `--verbose, -v` | Show detailed output | boolean | No |

For detailed information about global options, see [global options](/cli/global-options).

</Tab>
</Tabs>

### Examples
```bash Terminal
# Update regular variable
npx checkly env update ENVIRONMENT "staging"

# Update and convert to secret
npx checkly env update API_KEY "new-secret-key" --secret

# Update existing secret (must include --secret flag)
npx checkly env update DATABASE_PASSWORD "new-password" --secret
```

<Warning>When updating secrets, always include the `--secret` flag.</Warning>

## Variable Types

### Regular Variables
Standard key-value pairs visible to all team members:

```bash Terminal
npx checkly env add ENVIRONMENT "production"
```

### Locked Variables
Private to read-only users, visible to editors and admins:

```bash Terminal
npx checkly env add INTERNAL_API_URL "https://internal.api.com" --locked
```

### Secrets
Encrypted storage, values never displayed after creation:

```bash Terminal
npx checkly env add API_SECRET "super-secret-key" --secret
```

## Variable Scope

Environment variables managed by `checkly env` are **global** and available to:
- All checks in your account
- All check groups  
- All team members (unless locked)

For check-specific or group-specific variables, use the web UI or configure them directly in your monitoring-as-code setup.

## Best Practices

1. **Use secrets for sensitive data** like API keys, passwords, and tokens
2. **Lock internal variables** that shouldn't be visible to read-only users
3. **Use descriptive names** following standard conventions (UPPER_SNAKE_CASE)
4. **Document variables** and their purposes for team collaboration
5. **Regular cleanup** of unused variables to maintain security
