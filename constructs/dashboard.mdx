Use Dashboard to create public or private dashboards that display checks and their related metrics on a single page. Dashboards provide a centralized view of your monitoring data.

<Info>
Learn more about dashboards in the [Dashboards documentation](/docs/communicate/dashboards/).
</Info>

## `Dashboard` Examples

<CodeGroup>

```ts Basic Example
import { Dashboard } from 'checkly/constructs'

new Dashboard('basic-dashboard', {
  header: 'Service Status',
  description: 'Real-time monitoring dashboard',
  tags: ['production', 'api'],
  customUrl: 'service-status'
})
```

```ts Advanced Example
import { Dashboard } from 'checkly/constructs'
import * as path from 'path'

new Dashboard('advanced-dashboard', {
  header: 'ACME Production Status',
  description: 'Comprehensive monitoring for all ACME services',
  tags: ['prod', 'api', 'web'],
  logo: 'https://assets.acme.com/images/acme-logo.png',
  customUrl: 'acme-production-status',
  customDomain: 'status.acme.com',
  favicon: 'https://assets.acme.com/favicon.ico',
  link: 'https://acme.com',
  width: 'FULL',
  refreshRate: 60,
  paginate: true,
  paginationRate: 30,
  checksPerPage: 20,
  useTagsAndOperator: true,
  hideTags: false,
  enableIncidents: true,
  expandChecks: false,
  showHeader: true,
  showP95: true,
  showP99: true,
  isPrivate: false,
  customCSS: {
    entrypoint: path.join(__dirname, 'dashboard.css')
  }
})
```

</CodeGroup>

## `Dashboard` Configuration

<Tabs>
  <Tab title="Basic Configuration">

Configure essential dashboard settings:

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `header` | `string` | ❌ | - | Text displayed at the top of your dashboard |
| `description` | `string` | ❌ | - | Text displayed below the header |
| `tags` | `string[]` | ❌ | `[]` | Tags that filter which checks appear on the dashboard |
| `customUrl` | `string` | ❌ | - | Subdomain under "checklyhq.com" (required if customDomain not specified) |
| `customDomain` | `string` | ❌ | - | Custom domain (e.g., "status.example.com") |

</Tab>

<Tab title="Branding Configuration">

Configure visual branding and appearance:

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `logo` | `string` | ❌ | - | URL to logo image for the dashboard header |
| `favicon` | `string` | ❌ | - | URL to favicon image |
| `link` | `string` | ❌ | - | URL to redirect when dashboard logo is clicked |
| `customCSS` | `object` | ❌ | - | Custom CSS styling (Team/Enterprise plans only) |

</Tab>

<Tab title="Display Configuration">

Configure how the dashboard displays content:

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `width` | `string` | ❌ | `'FULL'` | Dashboard width: `'FULL'` \| `'960PX'` |
| `refreshRate` | `number` | ❌ | `60` | Auto-refresh interval: `60` \| `300` \| `600` seconds |
| `paginate` | `boolean` | ❌ | `true` | Enable pagination for checks |
| `paginationRate` | `number` | ❌ | `60` | Pagination interval: `30` \| `60` \| `300` seconds |
| `checksPerPage` | `number` | ❌ | `15` | Number of checks per page (1-20) |
| `useTagsAndOperator` | `boolean` | ❌ | `false` | Use AND instead of OR for tag filtering |
| `hideTags` | `boolean` | ❌ | `false` | Hide tags on the dashboard |
| `enableIncidents` | `boolean` | ❌ | `false` | Enable incidents (Team/Enterprise plans only) |
| `expandChecks` | `boolean` | ❌ | `false` | Expand checks by default |
| `showHeader` | `boolean` | ❌ | `true` | Show header and description |
| `showP95` | `boolean` | ❌ | `true` | Show P95 statistics |
| `showP99` | `boolean` | ❌ | `true` | Show P99 statistics |
| `isPrivate` | `boolean` | ❌ | `false` | Make dashboard private (Team/Enterprise plans only) |

</Tab>
</Tabs>

## `customCSS` Configuration

Configure custom CSS styling for your dashboard:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `entrypoint` | `string` | ❌ | Path to a CSS file containing custom styles |
| `content` | `string` | ❌ | Inline CSS content as a string |

<Info>
You must provide either `entrypoint` or `content`, but not both. Custom CSS is only available on Team and Enterprise plans.
</Info>

```ts
// Using file reference
customCSS: {
  entrypoint: path.join(__dirname, 'dashboard.css')
}

// Using inline content
customCSS: {
  content: `
    .header {
      background: #080808;
      border-bottom-color: #313035;
    }
    .header .logo a {
      color: #f7f8f8;
    }
  `
}
```

## Tag Filtering

Dashboards display checks based on tag filtering:

<Tabs>
  <Tab title="OR Filtering (Default)">
    ```ts
    new Dashboard('or-filtering-dashboard', {
      header: 'API or Web Services',
      tags: ['api', 'web'],
      useTagsAndOperator: false, // Default: shows checks with ANY of these tags
      customUrl: 'api-or-web'
    })
    // Shows checks tagged with 'api' OR 'web' OR both
    ```
  </Tab>
  
  <Tab title="AND Filtering">
    ```ts
    new Dashboard('and-filtering-dashboard', {
      header: 'Production API Services',
      tags: ['production', 'api'],
      useTagsAndOperator: true, // Shows checks with ALL of these tags
      customUrl: 'production-api'
    })
    // Shows only checks tagged with BOTH 'production' AND 'api'
    ```
  </Tab>
  
  <Tab title="All Checks">
    ```ts
    new Dashboard('all-checks-dashboard', {
      header: 'All Monitoring',
      tags: [], // Empty array shows ALL checks
      customUrl: 'all-monitoring'
    })
    ```
  </Tab>
</Tabs>

## Custom Domains

To use a custom domain for your dashboard:

<Tabs>
  <Tab title="DNS Configuration">
    ```bash
    # Add a CNAME record pointing to Checkly
    status.example.com CNAME checklyhq.com
    ```
  </Tab>
  
  <Tab title="SSL Certificate">
    Checkly automatically provisions SSL certificates for verified custom domains.
  </Tab>
  
  <Tab title="Configuration">
    ```ts
    new Dashboard('custom-domain-dashboard', {
      header: 'Service Status',
      customDomain: 'status.example.com', // Must be verified in Checkly UI
      tags: ['production']
    })
    ```
  </Tab>
</Tabs>

## Performance Optimization

<Tabs>
  <Tab title="Large Dashboards">
    ```ts
    // For dashboards with many checks
    new Dashboard('large-dashboard', {
      header: 'Comprehensive Monitoring',
      tags: ['all-services'],
      customUrl: 'comprehensive-status',
      refreshRate: 300, // Slower refresh for performance
      paginate: true,
      checksPerPage: 10, // Fewer checks per page
      paginationRate: 60
    })
    ```
  </Tab>
  
  <Tab title="High-Frequency Updates">
    ```ts
    // For critical dashboards needing frequent updates
    new Dashboard('critical-dashboard', {
      header: 'Critical Services',
      tags: ['critical'],
      customUrl: 'critical-status',
      refreshRate: 60, // Fast refresh
      paginate: false, // No pagination for immediate view
      checksPerPage: 20
    })
    ```
  </Tab>
</Tabs>

<Warning>
**Custom Domain Verification**: Custom domains must be verified through the Checkly UI before they can be used. DNS changes may take time to propagate.
</Warning>

<Info>
**Plan Limitations**: Some features like custom CSS, private dashboards, and incidents are only available on Team and Enterprise plans.
</Info>
